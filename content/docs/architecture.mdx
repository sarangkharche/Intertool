## Tech stack

- **Framework**: [Next.js 16](https://nextjs.org) with App Router, React 19, Turbopack
- **Auth**: [NextAuth v5](https://authjs.dev) with GitHub OAuth
- **Storage**: [AWS S3 SDK v3](https://docs.aws.amazon.com/AWSJavaScriptSDK/v3/latest/) (any S3-compatible service)
- **UI**: [shadcn/ui](https://ui.shadcn.com) + [Tailwind CSS 4](https://tailwindcss.com) + [Lucide](https://lucide.dev) icons
- **Docs**: [Fumadocs](https://fumadocs.vercel.app) with MDX
- **Fonts**: Inter (sans) + Geist Mono (monospace)
- **Editor**: [@uiw/react-md-editor](https://github.com/uiwjs/react-md-editor) for markdown with preview
- **CLI**: TypeScript with Commander.js, published as `intertool` on npm

## Data model

All skill data lives in your S3 bucket:

```
s3://your-bucket/
├── skills/{slug}/
│   ├── skill.json              # current version
│   └── versions/
│       ├── 1.0.0.json          # version snapshots
│       └── 1.0.1.json
├── mcp-servers/{slug}/skill.json
├── agent-tools/{slug}/skill.json
├── prompt-templates/{slug}/skill.json
├── _index.json                 # auto-rebuilt catalog
└── _categories.json            # seeded on first setup
```

## Skill types

| Type | S3 folder | Description | Auto-detected from |
|---|---|---|---|
| **Skill** | `skills/` | Claude Code skills | `SKILL.md`, `skill.yaml` |
| **MCP Server** | `mcp-servers/` | Model Context Protocol servers | `server.json` |
| **Agent Tool** | `agent-tools/` | Reusable tools for AI agents | `skill.yaml` |
| **Prompt Template** | `prompt-templates/` | Shared prompt patterns | `skill.yaml` |

## Access control

Intertool uses role-based access control (RBAC) with three roles:

| Role | Permissions |
|---|---|
| **Admin** | Full access: settings, members, tokens, publish, edit, delete |
| **Member** | Publish and edit own skills, browse and install all |
| **Viewer** | Browse and install only |

The first user or the user specified in `INTERTOOL_ADMIN` is assigned the admin role. New users are assigned the member role by default in self-hosted mode.

## SaaS mode

Intertool supports multi-tenant SaaS mode with subdomain-based organization routing. Each organization gets its own subdomain (e.g., `acme.intertool.sh`) with isolated settings and data stored in Upstash Redis.

SaaS mode is activated by setting `INTERTOOL_MODE=saas` and configuring Redis credentials.
